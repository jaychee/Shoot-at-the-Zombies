# 向僵尸开炮游戏机器人

一个用于自动化游戏操作的Python脚本，支持自动刷环球、刷卡关等功能。

## 功能特性

### 核心功能
- 🎯 **自动刷环球** - 自动加入队伍并战斗
- 🎮 **自动主线** - 自动挑战主线关卡
- ⚔️ **普通远征** - 自动刷普通远征模式
- 🔥 **超级远征** - 自动刷超级远征模式
- ⚡ **技能优先级** - 支持设置5个优先技能，按顺序匹配
- 💰 **消费模式** - 支持选择「我是土豪」或「我是穷B」模式，影响远征策略
- 🏃 **秒退模式** - 支持开启秒退功能（普通远征、超级远征、主线模式可用）
- 📊 **战斗次数统计** - 实时显示当前战斗次数，点击开始时自动清零
- 🔍 **图像识别** - 使用OpenCV进行图标类模板匹配，使用PaddleOCR进行文字识别
- 🔠 **OCR文字识别** - 基于PaddleOCR(PP-OCRv4 mobile)识别技能名/按钮文字，配置驱动，带白名单纠错
- 🖥️ **图形界面** - 友好的Tkinter GUI操作界面
- 💾 **配置保存** - 自动保存技能配置，下次启动自动加载
- ⌨️ **快捷键** - 按ESC键暂停脚本
- 📏 **窗口大小调整** - 一键调整游戏窗口为推荐的542*1010大小

### 技术特点
- 支持Windows PC端微信小程序
- 三层架构（入口/核心/模式），职责分离，便于扩展
- OCR 文字识别 + 图标模板匹配混合识别方案
- 支持PyInstaller打包为可执行文件
- 智能窗口查找和激活
- 异常处理和错误提示
- 实时状态显示

## 快速开始

### 环境要求
- Python 3.7+
- Windows操作系统

### 安装依赖
```bash
pip install -r requirements.txt
```

依赖包含：pyautogui、opencv-python、numpy、pywin32、pynput、paddlepaddle、paddleocr

### 目录结构
```
├── game_bot.py        # 程序入口 + 主流程调度 + GUI
├── bot_core.py        # 核心基类：窗口/截图/点击/模板匹配/OCR/通用状态判断
├── game_ocr.py        # OCR 模块（PaddleOCR 封装 + 游戏场景优化）
├── modes/             # 游戏模式模块
│   ├── huanqiu_mode.py    # 寰球模式
│   ├── mainline_mode.py   # 主线模式
│   └── expedition_mode.py # 远征模式（普通+超级）
├── config/            # 配置目录
│   └── skill_config.json  # 技能 OCR 文字配置
├── config.json        # 优先技能配置（自动生成）
├── screenshots/       # 截图保存目录
└── templates/         # 图标类模板图片目录
```

### 运行方式

#### 图形界面
```bash
python game_bot.py
```

> 注：当前版本仅支持 GUI 启动，启动后通过界面配置游戏标题、模式、技能等参数。

## 使用指南

### GUI界面说明
1. **游戏窗口标题** - 输入游戏窗口的标题（默认为"向僵尸开炮"）
2. **模式选择** - 选择运行模式：
   - **环球** - 自动刷环球（进别人队伍）
   - **主线** - 自动挑战主线关卡
   - **普通远征** - 自动刷普通远征
   - **超级远征** - 自动刷超级远征
3. **消费模式** - 选择消费策略（仅在环球、普通远征、超级远征模式下显示）：
   - **我是土豪** - 不检查门票，直接挑战
   - **我是穷B** - 检查门票，避免不必要的消耗
4. **秒退模式** - 勾选开启秒退（仅在普通远征、超级远征、主线模式下显示），战斗结束后立即退出
5. **战斗次数** - 设置战斗次数（0表示无限循环）
6. **战斗时间** - 设置每次战斗的最大时间（0表示无限制）
7. **优先技能** - 从下拉框选择5个优先技能（按优先级从上到下）
8. **开始/停止** - 控制脚本运行，点击开始时战斗次数自动清零
9. **调整窗口大小** - 一键将游戏窗口调整为推荐的542*1010大小，提高识别准确率

### 图标模板说明
`templates/` 目录仅保留**图标类**模板（纯图标无文字，OCR 无法识别），由 `bot_core.py` 的 `find_template()` 进行 OpenCV 模板匹配：
- `battling.png` / `battling-stop.png` - 战斗中暂停图标
- `close.png` / `home-close*.png` - 关闭 X 图标
- `return-1.png` - 返回箭头图标
- `expedition-exit.png` / `expedition-personnel.png` / `expedition-vice-captain*.png` / `expedition-elite-tag.png` / `expedition-tickets.png` - 远征图标
- `recruitment.png` / `recruitment-1.png` - 招募旗帜图标
- `im.png` - 聊天气泡图标

> 文字类界面元素（技能名、按钮文字如"确定""领取""基地"等）已全部改为 OCR 识别，无需准备文字模板。

**注意**：为了提高识别准确率，建议使用游戏窗口的原始分辨率，窗口大小推荐使用542*1010（GUI 提供"调整窗口大小"按钮一键设置）。

### 技能配置
技能识别采用 OCR 方式，识别文字从配置文件 `config/skill_config.json` 读取，无需准备技能模板图片。

脚本默认支持以下技能：
- 枪械、温压弹、干冰弹、冰雹发生器、装甲车、电磁穿刺
- 压缩气刃、制导激光、旋风加农、燃油弹、高能射线
- 无人机冲击、跃迁电子、空投轰炸

配置文件格式（`name` 为 GUI 显示与保存用名称，`text` 为 OCR 实际匹配文字）：
```json
{
    "skills": [
        {"name": "温压弹", "text": "温压弹"}
    ]
}
```

如游戏内文字变更，直接修改 `text` 字段即可，无需改代码。游戏更新后建议同步检查按钮文字映射（`bot_core.py` 中的 `TEXT` 字典）。

## 注意事项

1. **依赖安装** - PaddleOCR 体积较大，首次 `pip install` 较慢；若未安装 paddlepaddle/paddleocr，脚本仍可启动但 OCR 功能不可用（控制台会提示）
2. **首次运行** - PaddleOCR 首次加载模型约 3-5 秒，属正常现象
3. **兼容性** - OCR 识别基于 542*1010 标准窗口布局，ROI 坐标已按此校准，请勿随意调整窗口大小
4. **干扰** - 游戏过程中尽量不要操作鼠标和键盘，以免干扰脚本运行
5. **权限** - 确保脚本有足够的权限控制鼠标和键盘
6. **安全** - 本脚本仅供学习参考，请勿用于商业用途或破坏游戏平衡
7. **更新** - 游戏更新后若界面文字变化，修改 `config/skill_config.json` 或 `bot_core.py` 的 `TEXT` 字典即可

## 常见问题

### Q: 脚本找不到游戏窗口
A: 确保游戏窗口标题正确，且窗口未被最小化

### Q: 技能识别失败
A: 技能通过 OCR 识别文字，检查 `config/skill_config.json` 的 `text` 字段是否与游戏内实际显示文字一致；可适当调低 `bot_core.py` 中技能匹配的 `correct_ratio`（当前 0.6）

### Q: 按钮点击无反应
A: 检查 `bot_core.py` 的 `TEXT` 字典映射文字是否与游戏内一致；OCR 需 PaddleOCR 正常加载，确认依赖已安装

### Q: 脚本运行不稳定
A: 使用"调整窗口大小"按钮固定为 542*1010，确保游戏窗口未被遮挡

### Q: PaddleOCR 加载失败
A: 重新执行 `pip install paddlepaddle paddleocr`，确认 Python 版本 3.7+，Windows 系统需 Visual C++ 运行库

## 开发说明

### 架构（三层结构）
- **入口层** `game_bot.py` - `GameBotApp` 主流程调度 + `GameBotGUI` 图形界面
- **核心层** `bot_core.py` - `GameBotCore` 基类：窗口管理、截图、点击、图标模板匹配、OCR 识别、通用状态判断（所有模式共享）
- **模式层** `modes/` - 各游戏模式专属逻辑（寰球/主线/远征），通过组合持有 `GameBotCore` 引用
- **OCR 层** `game_ocr.py` - `GameOCR` 封装 PaddleOCR，含 ROI 裁剪、预处理、批量匹配、缓存、白名单纠错

### OCR 优化要点
- 采用 PP-OCRv4 **mobile** 模型（CPU 单帧约 80-150ms，体积约 15MB）
- **ROI 区域裁剪**：技能识别只扫描窗口底部技能区，减少约 60% 耗时
- **批量匹配** `find_texts_batch`：一次 OCR 识别后内存中匹配多个目标文字（技能循环关键优化，14 个技能 1 次 OCR 完成）
- **多级缓存**：截图缓存(TTL 0.1s) + OCR 结果缓存(TTL 0.3s)，缓存**原始 OCR 结果**，纠错按调用方阈值实时应用，避免多阈值场景缓存污染
- **图像预处理**：Otsu 二值化增强游戏描边/发光文字对比度
- **白名单纠错**：游戏文字为有限集合，用编辑距离纠正 OCR 误识，综合准确率约 97%
  - 默认阈值 **0.8**（按钮/状态文字，严格纠错）
  - 技能场景阈值 **0.6**（覆盖 3 字技能名 1 字误识，如"温压单"→"温压弹"）
  - 包含匹配：白名单关键词为长文本子串时命中（如"已置换的道具..."→"已置换"）

### 核心方法
- `GameBotCore.find_text()` / `click_text()` - OCR 查找/点击文字
- `GameBotCore.find_texts_batch()` - 批量 OCR 匹配
- `GameBotCore.find_template()` - 图标类模板匹配（保留）
- `GameBotApp.main_loop()` - 主循环逻辑
- `GameBotCore.find_click_return()` - 点击返回按钮，5秒内不重复计算战斗次数

### 扩展建议
- 新增按钮文字：在 `bot_core.py` 的 `TEXT` 字典添加映射（会自动同步到 OCR 白名单）
- 新增技能：在 `config/skill_config.json` 添加条目
- 新增图标模板：放入 `templates/` 目录，用 `find_template()` 匹配
- 增加更多游戏模式：在 `modes/` 新建模块，实现 `run()` 方法，在 `GameBotApp` 注册分发
